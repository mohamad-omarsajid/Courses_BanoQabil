    Glossary¶

    Every technical term used in this book, in plain English with an Urdu note where it helps. Terms are added here as each lesson is written. Use the search box to jump to a term, or browse A to Z below.

    A¶

    active learning¶

    Learning by doing the thing yourself, not just watching or reading. First introduced in lesson 0.4.

    Roman Urdu: khud kar ke seekhna

    active recall¶

    Pulling an answer from memory instead of re-reading it. It builds strong memory. First introduced in lesson 0.4.

    Roman Urdu: yaad se jawab nikalna, dobara parhne ke bajaye

    absolute URL¶

    A full web address that starts with https and points to another website. First introduced in lesson 2.2.

    Roman Urdu: poora pata jo https se shuru hota hai

    accessibility¶

    Making a page usable for everyone, including people with disabilities. First introduced in lesson 2.4.

    Roman Urdu: safhe ko sab ke liye qabil-e-istemal banana

    alt text¶

    Text that describes an image for screen readers, search engines, and broken images. First introduced in lesson 2.2.

    Roman Urdu: tasveer ki tafseel wala likha

    anchor¶

    The <a> tag that creates a link to another page or site. First introduced in lesson 2.2.

    Roman Urdu: link banane wala tag

    attribute¶

    Extra information inside an opening tag, written as name="value". First introduced in lesson 2.1.

    Roman Urdu: tag ke andar di gayi maloomat

    align-items¶

    A flexbox property that aligns items along the cross axis. First introduced in lesson 3.2.

    Roman Urdu: items ko cross axis par set karta hai

    @apply¶

    A CSS rule that puts several Tailwind utilities into one custom class. First introduced in lesson 5.2.

    Roman Urdu: kai Tailwind classes ko ek class mein rakhne ka tareeqa

    arbitrary value¶

    A custom Tailwind value written inside square brackets, like bg-[#0fab95]. First introduced in lesson 5.3.

    Roman Urdu: square brackets mein likhi gayi custom Tailwind value

    AbortController¶

    A browser tool that can cancel a running fetch request. First introduced in lesson 11.2.

    Roman Urdu: chalti hui fetch ko rokne wala tool

    API¶

    A web address that returns data your app can use, usually as JSON. First introduced in lesson 11.2.

    Roman Urdu: aik web address jo data wapas karta hai

    argument¶

    The real value you pass when you call a function. First introduced in lesson 7.3.

    Roman Urdu: asal value jo aap function call karte waqt dete hain

    arrow function¶

    A short way to write a function using the => arrow. First introduced in lesson 7.3.

    Roman Urdu: function likhne ka chhota tareeqa jo => arrow se hota hai

    App Router¶

    The Next.js routing system that uses the app folder, where folders become routes. First introduced in lesson 15.1.

    Roman Urdu: app folder wala routing system

    AnimatePresence¶

    A Framer Motion wrapper that lets a component play an exit animation before it unmounts. First introduced in lesson 16.1.

    Roman Urdu: jane se pehle exit animation chalane wala wrapper

    AmbientLight¶

    A Three.js light that lights everything evenly with no shadows. First introduced in lesson 17.2.

    Roman Urdu: har taraf se halki roshni

    AnimationMixer¶

    A Three.js object that controls playback of a model's animation clips. First introduced in lesson 17.3.

    Roman Urdu: model ki animation chalane wala

    animation budget¶

    The small limit of motion a page should use before it feels heavy or slow. First introduced in lesson 16.4.

    Roman Urdu: ek page par jaayaz haraka ki hadd

    Anime.js¶

    A small, free JavaScript animation library, good for self-contained effects and SVG drawing. First introduced in lesson 16.2.

    Roman Urdu: chhoti animation library

    analytics¶

    Data about who visits your site and which pages they see. First introduced in lesson 19.4.

    Roman Urdu: maaloomat ke kitne log aaye aur kaun se pages dekhe

    any¶

    A TypeScript type that turns off all checks for a value. Avoid it. First introduced in lesson 18.1.

    Roman Urdu: saare checks band kar dene wali type

    assertion¶

    A test line that checks if a value matches what you expect. First introduced in lesson 18.3.

    Roman Urdu: aik line jo value ko expected se milati hai

    array¶

    An ordered list of values stored together. First introduced in lesson 8.1.

    Roman Urdu: cheezon ki list

    B¶

    backend¶

    The part of a website that runs on a server and handles data, accounts, and logic. Users never see it. First introduced in lesson 1.1.

    Roman Urdu: website ka woh hissa jo server par chalta hai aur data sambhalta hai

    browser¶

    A program that turns website code into the page you see, like Chrome or Firefox. First introduced in lesson 0.1.

    Roman Urdu: woh app jis se tum websites dekhte ho

    block element¶

    An element that starts on a new line and takes the full width, like a paragraph. First introduced in lesson 2.2.

    Roman Urdu: nayi line se shuru hone wala element

    body¶

    The part of an HTML page that holds the content users see on screen. First introduced in lesson 2.1.

    Roman Urdu: page ka wo hissa jo user screen par dekhta hai

    box model¶

    The idea that every element is a box of content, padding, border, and margin. First introduced in lesson 3.1.
