# Chapter 13: The ecommerce capstone

<p class="byline" markdown>Muhammad Umar Sajid · Trainer at Bano Qabil Sahiwal · Front End Development and Graphic Design</p>

This is the big one. You will build a full ecommerce store from start to finish.
It uses everything you have learned: React, Tailwind, state, forms, and GSAP. By
the end you will have a real, shippable project with a live URL.

The store has five pages: a home page, a shop page, a product detail page, a cart,
and a checkout. You build it in four phases, one per lesson. We model it on a real
reference project, the BQ-Store-FED store, so you can compare your work to a
finished example.

Reference project: [BQ-Store-FED on GitHub](https://github.com/mohamad-omarsajid/BQ-Store-FED).

!!! tip "This is your portfolio piece"
    A finished store is the project that wins junior-dev interviews. Take your time,
    build each phase fully, and keep the code clean. You will link to this for years.

## Mega assignment

!!! bq-assignment "Ship a complete, real ecommerce store"
    This is the capstone the whole course has pointed at. Build a full store you
    would be proud to put on a CV and send to a client. Theme it around real
    products: a Sahiwal clothing brand, a bookshop, a dates-and-honey seller,
    whatever you would actually enjoy building.

    **What you build**

    - Five working pages: home, shop (with filtering and search), product detail, cart, and checkout.
    - A cart that **remembers its contents** across page reloads and shows a live total.
    - A checkout form with full validation and an order-confirmation screen.
    - Tasteful animation, a responsive layout that works on a real phone, and the store deployed live.

    **Done when**

    - [ ] All five pages work and are linked by routing, with no dead ends.
    - [ ] Adding, changing quantity, and removing cart items updates the total correctly.
    - [ ] The cart survives a page refresh.
    - [ ] Checkout validates input and shows a clear confirmation.
    - [ ] The store is live on a public URL and works on mobile.
    - [ ] The README explains the project, the stack, and the live link, with screenshots.

    **Stretch goal:** Add a wishlist, a "you might also like" row on the product page, and an empty-cart state with a friendly nudge back to the shop.
