# Chapter 7: JavaScript fundamentals

<p class="byline" markdown>Muhammad Umar Sajid · Trainer at Bano Qabil Sahiwal · Front End Development and Graphic Design</p>

So far your pages sit still. HTML gives them structure, CSS gives them a look,
but they do not react. JavaScript changes that. It is the language that makes a
page think, respond to clicks, and update on its own.

This is the chapter many learners find hardest. That is normal. Go slowly, type
every example, and run it yourself. You learn JavaScript by writing it, not by
reading it. Each lesson builds a small real thing you can run today.

## Lessons in this chapter

- [ ] [7.1 Variables, types, and operators](7-1-variables-types-operators.md): store values, learn the basic types, and do math and comparisons.
- [ ] [7.2 Conditionals and loops](7-2-conditionals-and-loops.md): make choices and repeat work without copying code.
- [ ] [7.3 Functions](7-3-functions.md): wrap code into named tools you can reuse anywhere.
- [ ] [7.4 Higher-order functions and error handling](7-4-higher-order-and-errors.md): pass functions around and handle things that go wrong.

!!! tip "Run it in the console"
    You do not need a project to try JavaScript. Open any web page, press F12 to
    open DevTools from lesson 1.4, and click the Console tab. Type code there and
    press Enter to see it run at once.

## Mega assignment

!!! bq-assignment "Add a working tool to your client site with plain JavaScript"
    Your page can now think. Add one feature the business actually needs, using
    nothing but the JavaScript from this chapter. No libraries, no frameworks.

    **What you build** (pick the one that fits your client)

    - A **price or quote calculator**: the customer picks options and sees a total update live (a tailor's stitching quote, a caterer's per-head cost, a salon package).
    - Or a **tip and bill splitter** for a caf&eacute; or restaurant.
    - Or a **short quiz** that gives a result at the end (for a tutor or a coaching centre).

    Use variables and types, conditionals, a loop, and at least one function you wrote yourself.

    **Done when**

    - [ ] The feature updates the page when the user changes an input.
    - [ ] You used a function, a conditional, and a loop, each for a real reason.
    - [ ] Bad input (empty or zero) is handled, not crashed on.
    - [ ] There are zero red errors in the Console.

    **Stretch goal:** Handle the error cases with `try`/`catch` and show the user a friendly message instead of a broken number.
