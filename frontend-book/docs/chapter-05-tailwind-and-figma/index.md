# Chapter 5: Tailwind CSS and Figma

<p class="byline" markdown>Muhammad Umar Sajid · Trainer at Bano Qabil Sahiwal · Front End Development and Graphic Design</p>

You can now write CSS by hand. That is a skill worth having. But on real jobs,
most teams move faster with Tailwind CSS. Tailwind lets you style right inside
your HTML with small, clear class names. No more jumping between two files.

This chapter also teaches the other half of real work: reading a design. Clients
and designers hand you files made in Figma. You learn to open one, read its
colours and sizes, and turn it into a working page.

- [ ] [5.2 Building real components with Tailwind](5-2-building-real-components.md): cards, buttons, a navbar, and a hero, the real building blocks.

!!! tip "Two skills at once"
    By the end of this chapter you can both style fast with Tailwind and read a
    designer's file. Put together, that is most of what a junior frontend job asks
    for on day one.

## Mega assignment

!!! bq-assignment "Rebuild a section from a Figma design with Tailwind"
    Real jobs hand you a Figma file and expect a faithful build. Practise that
    exact workflow on your client site.

    **What you build**

    - Take one section of your client site (the hero, or a pricing or menu card grid). Design it in Figma first, or start from a free community design. Then rebuild it with Tailwind utility classes, matching the spacing, colours, and font sizes by reading the real values out of Figma.

    **Done when**

    - [ ] The build matches the design's spacing and colours within a few pixels.
    - [ ] You styled with Tailwind utilities, not a separate hand-written stylesheet.
    - [ ] It is responsive using Tailwind's `sm:` `md:` `lg:` prefixes.

    **Stretch goal:** Pull the repeated pattern (say, a card) into one reusable block so you are not copying the same long class list five times.
