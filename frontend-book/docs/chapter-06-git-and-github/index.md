# Chapter 6: Git and GitHub

<p class="byline" markdown>Muhammad Umar Sajid · Trainer at Bano Qabil Sahiwal · Front End Development and Graphic Design</p>

You can build pages now. But how do you save your progress safely, go back when
you break something, and share your work with the world? That is what Git and
GitHub do. Every real developer uses them every day.

Git tracks the history of your code on your own computer. GitHub stores a copy
online, so you can back it up, show it to others, and work as a team. This
chapter takes you from your first commit to a live site on the internet.

## Mega assignment

!!! bq-assignment "Put your client site on GitHub, live, with a real workflow"
    Code that lives only on your laptop is code nobody can see. Push your client
    site to GitHub the way a team would, and put it on the internet for free.

    **What you build**

    - A GitHub repository for your client site, with clean, meaningful commit messages (not "stuff" and "final final v2").
    - A real branch workflow: make a change on a branch, open a pull request against `main`, review your own diff, then merge.
    - The site live on **GitHub Pages**, on a URL you can text to anyone.
    - A `README.md` that says what the site is, who it is for, and the live link.

    **Done when**

    - [ ] Your repo has at least five commits with clear messages.
    - [ ] You created a branch, opened a pull request, and merged it.
    - [ ] The site loads on its live GitHub Pages URL.
    - [ ] The README explains the project and links the live site.

    **Stretch goal:** Add a `.gitignore`, and send the live link to the business owner for feedback.
