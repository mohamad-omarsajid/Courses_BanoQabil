# Contributing

Thank you for helping improve Bano Qabil Sahiwal Courses. This project is for beginners, so clarity matters more than cleverness.

## Ways To Contribute

- Fix typos, broken links, confusing wording, or layout issues.
- Improve lesson explanations in plain English or Roman Urdu.
- Add beginner-friendly examples, diagrams, or screenshots.
- Report bugs in the landing page or course pages.
- Suggest missing topics through GitHub Issues.

## Local Development

```bash
npm --prefix web install
npm run dev
```

For a production check:

```bash
npm run build
```

## Content Style

When editing course lessons:

- Write for complete beginners.
- Explain new terms the first time they appear.
- Prefer small steps over large jumps.
- Use examples that feel familiar to students in Pakistan when possible.
- Keep Roman Urdu natural, simple, and WhatsApp-friendly.
- Avoid overpromising jobs, income, certificates, or outcomes.

## Code Style

- Follow existing React and TypeScript patterns in `web/src`.
- Keep UI changes responsive on mobile and desktop.
- Do not add a new dependency unless it clearly solves a real problem.
- Keep generated build output out of commits unless it is intentionally placed in `web/public`.

## Pull Request Checklist

Before opening a pull request:

- Run `npm run build`.
- Check the affected page in a browser.
- Confirm links still work.
- Add screenshots for visual UI changes.
- Keep the PR focused on one change or theme.

## Issue Guidelines

When opening an issue, include:

- What page or file has the problem.
- What you expected to happen.
- What actually happened.
- A screenshot if the issue is visual.
- Device and browser details for layout bugs.

## Community Standard

Be patient, respectful, and beginner-friendly. Many contributors and learners may be new to GitHub, coding, design, or English technical writing.
