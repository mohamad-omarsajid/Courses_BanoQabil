The terminal

Windows-first, with WSL2 Hands-on exercises Updated 26 May 2026

You have clicked folders open with a mouse your whole life. The terminal does the same job by typing. It feels slow at first. Then it becomes the fastest way to move around your computer. Developers live in it.

Aap ne saari zindagi mouse se folders open kiye hain. Terminal wohi kaam typing se karta hai. Pehle slow lagta hai. Phir yehi sab se tez tareeqa ban jata hai apne computer mein move karne ka. Developers iss mein hi rehte hain.

What you'll know by the end

  • What a terminal is, and why developers use it
  • The different terminal apps on Windows and macOS, and which to use
  • How to read the prompt, the text already sitting there
  • The everyday commands for moving and making files

What a terminal is

A terminal is a window where you type commands instead of clicking. You type one line, press Enter, and the computer does it. The program that reads your line and runs it is called a shell.

Terminal ek aisi window hoti hai jahan aap click karne ke bajaye commands type karte hain. Ek line type karo, Enter dabao, aur computer woh kaam kar deta hai. Jo program aap ki line parh kar use chalata hai use shell kehte hain.

Why bother, when clicking works? Three reasons. It is faster once you know it. It works the same on every machine. And almost every developer tool, like Git and Node, expects you to use it.

Toh pareshani kyun, jab clicking se kaam ho jata hai? Teen wajahein hain. Pehli baat, jab seekh jao to ye tezi se hota hai. Doosri baat, har machine pe ek jaisa kaam karta hai. Aur teesri baat, har developer tool, jaise Git aur Node, aap se ye hi expect karte hain.

Reading the prompt

When you open the terminal, there is already some text sitting there, ending in a $. That is the prompt, and it is not random. It tells you who and where you are. A typical Linux prompt looks like this:

your terminal bash Copy 
ali@laptop:~/projects$

Here is what each part means:

PartWhat it means
aliYour username on the computer
@laptopThe name of the computer
~Your home folder, your starting place
~/projectsThe projects folder inside home
$The prompt symbol. You type right after it.

So ali@laptop:~/projects$ simply says: "Ali, on the laptop, currently inside the projects folder, ready for a command." After you type and press Enter, the answer prints below, and a fresh prompt appears.

Yani ali@laptop:~/projects$ simply ye keh raha hai: "Ali, laptop pe, abhi projects folder ke andar hai, aur command ke liye taiyaar hai." Type karne aur Enter dabane ke baad, jawab neeche print ho jata hai, aur ek nayi prompt aa jati hai.

Which terminal app should I open?

Different systems give you different terminal apps. Pick your tab.

Windows
macOS
Linux

Windows actually has a few, which confuses beginners:

  • Command Prompt (cmd) is the old, basic one. It works, but is limited.
  • PowerShell is Microsoft's newer, more powerful shell.
  • Windows Terminal is a modern app that holds all of them in tabs. If you do not have it, install it free from the Microsoft Store.
  • For this course you use the Ubuntu (bash) shell from WSL.

On macOS, you have two solid choices:

  • Terminal.app ships with every Mac. Open Spotlight, type "Terminal".
  • iTerm2 is the popular free upgrade with tabs and themes.

Most Linux desktops ship with a terminal already. GNOME Terminal, Konsole, and xterm are all fine. You're already on the right team.

The everyday commands

Here is the short list you will use every day. Type them, press Enter, see what happens. Don't memorise. You will know these by Chapter 3 because you will have typed each one a hundred times.

your terminal bash Copy 
# where am I?
pwd

# list what is in this folder
ls

# move into a folder
cd projects

# go back up one folder
cd ..

# make a new folder
mkdir my-site

# make a new empty file
touch index.html
A small trick

Hit Tab while typing a file or folder name. The terminal will auto-complete it for you. This single trick will save you an hour a week.

Read an error message calmly

Errors look scary. They are not. An error is the computer telling you exactly what stopped it. The trick is simple: read the last line first.

bash 
# try to enter a folder that does not exist
cd projcts
bash: cd: projcts: No such file or directory

Read the last part: "No such file or directory". The folder name is wrong. You typed projcts instead of projects. Fix the spelling and try again. Most errors are this small.

Patience pays here

Every developer reads error messages they wrote themselves, every day, forever. Getting calm at this is half of the skill. The Prophet ﷺ said patience is light, and nowhere is that more obvious than at 1am in front of a red error.
